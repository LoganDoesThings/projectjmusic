# Security Policy

## Supported Versions

Currently, the following versions of JMusic are supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | ✅ Yes             |
| 1.0.x   | ❌ No              |
| < 1.0.0 | ❌ No              |

## Reporting a Vulnerability

We take the security of JMusic seriously. If you believe you have found a security vulnerability, please do not report it via public GitHub issues. Instead, please follow the steps below:

1. **Private Reporting:** Please send an email to **logan.motta@icloud.com**  with the subject line `SECURITY VULNERABILITY: [Brief Description]`.
2. **Details to Include:**
   - A detailed description of the vulnerability.
   - Steps to reproduce the issue (PoC).
   - Potential impact if exploited.
3. **Acknowledgment:** You will receive an acknowledgment of your report within 48-72 hours.
4. **Resolution:** We will work to investigate and resolve the issue as quickly as possible. We ask that you maintain confidentiality until we have had a chance to provide a fix for the community.

## Disclosure Policy

- We follow a "coordinated disclosure" model.
- We will not take legal action against researchers who report vulnerabilities according to this policy and provide a reasonable amount of time for a fix to be implemented before public disclosure.
- We will credit you in the release notes of the fixed version if you wish.
